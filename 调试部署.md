2. 部署 OpenIM 服务器 （IM）
   2.1 📡 设置OPENIM_IP
   🔔 必须设置OPENIM_IP，不能是 127.0.0.1

# If the server has an external IP
export OPENIM_IP="external IP"

# If providing only internal network services
export OPENIM_IP="internal IP"

2.2 🏗️ 部署组件（mongodb/redis/zookeeper/kafka/MinIO 等）
git clone https://github.com/OpenIMSDK/open-im-server && cd open-im-server
# It is recommended to switch to the release-v3.6 or later release branches

chmod +x scripts/*.sh

make init && docker compose up -d


2.3 🛠️ 编译
make build


2.4 🚀 启动/停止/检查
# Start
make start

# Stop
make stop
# Check
make check

6. 关于配置项目
   6.1 🛠️ 修改常用配置项
   配置项	待修改文件	行动
   mongo/kafka/minio 相关	.env，openim-server/config/config.yaml	需要重启组件和 IM
   redis/zookeeper 相关	.env、openim-server/config/config.yaml、聊天/config/config.yaml	需要重启组件、IM 和聊天
   秘密	openim-server/config/config.yaml、聊天/config/config.yaml	需要重启 IM 和 Chat
   6.2 🔄 通用配置中的特殊项目
   特殊配置项：API_OPENIM_PORT/MINIO_PORT/OPENIM_IP/GRAFANA_PORT

修改文件中的特殊配置项.env
根据规则修改openim-server/config/config.yaml中的配置
object:
apiURL: "http://$OPENIM_IP:$API_OPENIM_PORT"
minio:
endpoint: "http://$DOCKER_BRIDGE_GATEWAY:$MINIO_PORT"
signEndpoint: "http://$OPENIM_IP:$MINIO_PORT"

grafanaUrl: http://$OPENIM_IP:$GRAFANA_PORT

根据规则修改chat/config/config.yaml中的配置
If IM is not on the same machine as Chat, replace 127.0.0.1 with the IP of the machine where IM is located
openIMUrl: "http://127.0.0.1:$API_OPENIM_PORT"


重启 IM 和聊天
6.3 🛠️ 修改其他配置项
对于 .env、chat/config/config.yaml 和 openim-server/config/config.yaml 中的其他配置项，您可以在相应的文件中单独修改这些项。

6.4 修改端口
特别注意，任何与IM相关的端口修改也需要与open-im-server/scripts/install/environment.sh 中的端口同步
